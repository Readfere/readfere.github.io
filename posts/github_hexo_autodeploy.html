用GitHub Actions自动化部署Hexo博客

2023-12-04T01:26:00.000Z2023-12-04T03:33:14.681Z

以下是用 solitude 主题部署 hexo 的 GitHub Actions 的 yml 文件：

前置条件：

  1. 要新建一个私有仓库 hexo_blog 来存放未解析前的 Hexo 博客文件夹
  2. 然后把所有文件拷贝进去
  3. 一定要把 package-lock.json 文件拷贝进去，这个文件是记录依赖安装的，没有的依赖会通过它重新安装
  4. 安装依赖的时候出现了 hexo offline 错误提示：

    说明：hexo-offline 是 solitude 主题中用于设置 PWA（渐进式 web app）的，不是刚需
    解决方法：删除根目录里面 package.json 中关于 offline 的记录
  5. 生成静态文件时出现错误
    解决办法：在根目录建立 gulpfile.js 文件，并写入内容

说明：这个插件是用于对代码进行压缩
参考：进阶配置 🚀 | Solitude 主题官方文档

改完以上两项之后，autodeploy 成功运行，以后对仓库任何文件的增删改查都会自动部署到 GitHub Pages 中，简直太方便了
